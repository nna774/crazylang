ラムダ計算とは

導入された理由
"関数"を一般的に扱いたい

じゃあ関数ってなんだ？


  • 謎のブラックボックス
  • 引数(入力) がいくつかあって, 返り値(出力) のあるもの
  • "副作用"のある関数は今は置いときます
    副作用ってなんだよ
    後でやります……

普通の関数

f(x) = x + 2

ラムダ記法

λx. x + 2

赤が入力, 緑が出力

代入はこんな感じ

普通の関数

f(42) = 42 + 2 = 44

ラムダ記法

(λx. x + 2) 42 = 42 + 2 = 44
"λ(変数名)." の変数名を, ピリオドの後ろから探してきて, そこを引数で置き換える

わかりづらいですね, いくつか例を行きましょう
(λx. x) 42 = ？
42
(λx. (λy. x * y)) 42 = ？
(λy. 42 * y)
(λy. 42 * y) 63
42 * 63 = 2646
(λx. (λy. x * y)) 42 63 = ？
42 * 63 = 2646
これいいですか
一気に適用(代入を適用)できる

そんなふうに書いて何がうれしいんですか


今f って名前が元の関数にはついてました.
ラムダ記法ではどっかいってる.
2 をたす為の関数に名前いらない(今回は)
いちいち名前つける必要がない

他の利点

関数に関数を渡したり, 関数が関数を返したりできる

何を言っとるんや

例えば,
2引数を取って, 常に一つ目の値を返す関数
Lazy K のK関数です

k(x,y) = x

なんかそれっぽい

でもちょっと不便
k_2(y) = 42

kとめっちゃ似てるのにいい感じにできない
k(42,y) = 42

これは関数って言っていいのか
なんか気持ち悪い
ラムダ式で書くと
λx. (λy. x)

省略記法として
λxy. x
とも書く
λxy. x
これって2引数の関数っぽいですよね
もちろんそれは正しいです
でも,
一つ適用してみる
(λxy. x) 3 = (λy. 3)
λxy. x は λx. λy. x の省略だということを思いだしてください
ひとつ引数を消費して1引数の関数になりました
「一つ引数を取って, 「1引数の関数」を返す関数」 とも見れる
さっきのk に1引数を与えたもの
k(42,y) = 42
これを
k(42)(y) = 42
こう変換したような感じ
適当な関数をラムダ記法で書くだけで自動的にいい感じに変換される

一般化したらこんな感じ

f(x_0, x_1, .. , x_n) -> f(x_0)(x_1)..(x_n)
こんな感じの変換

2引数の場合だけだと

f(x,y) -> f(x)(y)
へとかわる

この変換は何が嬉しいのですか

さっきみたいに, 2引数以上の関数に, ひとつだけ引数を渡したりできる
"部分適用"という

一応形式的な定義


  • <式> := <識別子>
  • <式> := (λ<識別子>. <式>)
  • <式> := (<式> <式>)

これでできるもののみが式である

あんまりどうでもいいです(暴言)

簡単なパズルと考えましょう.
こんなのがラムダ式です
  • λx. x
  • λxy. x
  • λw. w
  • λx. x x

λxy. x はλx. λy. x の省略
1つ目と3つ目は同じラムダ式
